# _plans/ — Temporary Planning Files

This directory holds **short-lived working plans** that map 1:1 to GitHub Issues.

## Rules
- Each plan file is linked from its issue body.
- Plan files are **deleted in the same PR that closes the issue**.
- Permanent documentation lives in `AGENTS.md` only (the agent brain).
- Never add a plan file without an open issue.

## Format
Each file MUST contain:
1. Goal (1-2 lines)
2. In/Out of scope
3. Implementation checklist
4. Acceptance criteria
5. Files affected
6. Test plan
7. Cleanup note at the bottom

This directory is expected to be **empty most of the time**.
