# Incident: Balance Reads 125€ Instead of 2.23€

**Date**: 2026-05-07 | **Severity**: P2 | **Duration**: 10 minutes
**Agent**: stealth-orchestrator | **Component**: survey-cli/survey/scanner.py

## Symptoms
- `read_balance()` returned 125.00€
- Actual balance: 2.23€ (verified via CDP Runtime.evaluate)
- Balance displayed as 125€ in logs and agent output
- Led to incorrect "earnings" calculations

## Root Cause
`read_balance()` scanned entire `document.body.innerText` for numbers near € symbol, then took Math.max() of all matches:
```javascript
var valid = matches.filter(function(v) { return v > 0.01; });
return valid.length > 0 ? Math.max.apply(null, valid).toString() : "0";
```
The dashboard's Level progress bar showed "125" near a € sign. Math.max picked 125 instead of 2.23.

## Detection
```javascript
// Debug output showed all matched values:
// [0.03, 0.31, 0.21, 0.09, 0.58, 0.23, 0.40, 1.65, 0.97, 0.37, 0.55, 0.21, 125.0]
// Math.max → 125.0 (Level progress, NOT balance!)
```

## Fix
```javascript
// Changed from Math.max to context-aware filtering:
var lines = document.body.innerText.split("\\n");
for (var i = 0; i < lines.length; i++) {
    var line = lines[i].trim();
    var m = line.match(/^(\\d+[.,]\\d+)\\s*€?$/);
    // Skip lines that are survey rewards (have "Min" in adjacent lines)
    if (m && !/Min|Level|Umfragen/.test(lines[Math.max(0,i-1)] || "")) {
        var val = parseFloat(m[1].replace(",", "."));
        // Balance is typically > 1.0 and < 1000
        if (val > 1.0 && val < 1000) return val.toString();
    }
}
```

## Lessons
1. Never use Math.max() on uncontextualized financial data
2. Always filter by plausible value ranges
3. Check adjacent text lines for context keywords
4. Dashboard has multiple € values — survey rewards, balance, level progress
